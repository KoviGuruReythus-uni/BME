#### Breaking down the code
- `qsort` has to work for all var types, not just double, int... everything. So it uses addresses instead of pure values. 
- We copy the addresses to a new set of pointers *da *db 
- We use `const` to keep the original data safe
- Only when there is a comparison like `<, >, ==` do we look at the actual value at the addresses.
- We send `void` and then convert to double. This make it so that the compiler actually knows what var type we are comparing. This is also for `Type Safety`
	- Send `void` pointers is like sending a blind address, we don't know what is present there, it could be a house, a restaurant... anything. But it is not fair to compare a house and a school, we fucking cant. This is type safety. 
	- It also ensures that we are comparing say 8 bytes of data from both. int might by 4 bytes, some custom var type might be 40, cant be comparing those now can we? 

#### Function Pointer
- `qsort` is a sorting engine, but it doesn't inherently have anything idea based on what it has to sort the data. 
- The _function pointer_ is what tells it what algorithm to use in order to sort the data we got. 
	- In our case it is the _Compare_ function. 

#### Why void type vars are used? 
- `qsort` is generic
- It needs to be able to work with all types of vars. 
- Using void and then later specifying the type keeps it this way. 

#### Scanf
- `scanf("%f", &d[i]` why is it `&d[i]` and not just `d[i]`
- In C, every function has its own _stack frame_ it is like their own personal workspace. Think of every command like `scanf` `printf` to be custom functions written by you. 
	- We know that when we use some int in a function, it dies after that function is done running. 
	- That is the same for their personal workspace as well. Once the job of the function is done, that is erased and the memory is returned.
	- When we tell `scanf` to read a float, we need to tell it _where_ to read it from. 
	- Giving it a pointer does exactly that. It gives scanf an address that it can read from. 
		- This is like giving `scanf` _remote access_ to that var
		- If we just give it that var, it would most likely just be uninitialised garbage value.
