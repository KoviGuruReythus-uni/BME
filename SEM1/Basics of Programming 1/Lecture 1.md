### Imperative programming:
* We tell the computer step-by-step what to do with proper instructions and by _defining an algorithm_ 
* Steps taken during the course 
	* describe the task
	* construct an algorithm to solve this task
	* create the program - (coding)
* More detailed version, 
	* describe the task
	* give an exact spec of the task
	* select the right data structure for modelling the problem
	* construct an algo for solving the task
	* select and effective prog lang for coding (this course is C)
	* create the code of the algo
	* test the prog

### Algorithm:
A finite seq of steps, that can be performed mechanically and leads to the solution. 

Before coding there a couple of things to do,
1. check if the algorithm is _right
	* if it gives the req sol to our prob
2. check if it is _complete_
	* if it give sols to all possible cases of the problem
3. check if it is _finite_
	* if it ends in finite number of steps (no infinite loops)

* Even if the algo is right, complete and finite, it might not be manageable.
* It is important for the algorithm to be practically finite, 
	* End in a reasonable amount of time 
	* Use reasonable amount of data
